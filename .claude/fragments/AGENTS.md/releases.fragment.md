**Releases.** Tagging `v*` triggers `.github/workflows/release.yml`, which forwards to an exact-commit-pinned shared `release-swift.yml` workflow: a universal (arm64 + x86_64) `swift build`, mandatory Developer ID signing and notarization, a caller-owned draft verified before publication, and a Homebrew binary cask pushed to `yasyf/homebrew-tap`. The version comes from the tag, and the executable product must keep the repo's name — that's the whole calling contract. The release refuses to run unless the tagged commit is on `main` — tag a merged commit (e.g. `git tag vX.Y.Z origin/main`), not a feature branch. One-time setup requires `HOMEBREW_TAP_TOKEN` plus all `MACOS_*` signing and notarization secrets; missing credentials fail before packaging or publication.
